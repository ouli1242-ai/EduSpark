# 前端 UI 重构需求

> 后端核心功能全部完成后统一重写，此文档记录需求备忘。

## 总体方向

目标：不懂代码的外行人看到就觉得"高级、有科技感"。
不是花里胡哨的颜色堆砌，而是**质感、动效、布局细节**上的用心。

## 登录/注册页

现状问题：
- 紫色渐变背景 + 居中白色卡片，一眼就是 AI 项目模板
- Element Plus 默认组件风格，没有辨识度

期望方向：
- 有创意的背景动效（Canvas / CSS 粒子、网格光效、动态渐变等）
- 登录卡片的设计要有独特性，不一定是经典的居中卡片
- 过渡动画（页面切入、表单切换、成功反馈等）
- 品牌感（EduSpark 的 logo、字体、视觉语言）

## 整体布局与交互

现状问题：
- 侧边栏 + 顶栏 + 内容区，标准后台管理样式，太普通
- 页面切换没有过渡，生硬
- 整体缺少"呼吸感"（间距、留白、微动效）

期望方向：
- 布局可以有区分度，不一定是传统的左-顶-内容三段式
- 微交互动效（hover、切换、加载、消息出现等）
- 流畅的页面过渡动画
- 舒适的间距和排版，让人看着就舒服
- 自定义组件，减少 Element Plus 原生感

## 具体页面

### 智能对话页
- 消息气泡的样式可以更有设计感
- 输入框区域可以有更好的视觉反馈
- Agent 状态展示可以更直观（现在只是一行文字 + 进度条）

### 学习画像页
- 雷达图 + 详情描述，现在太简陋
- 可以考虑更有视觉冲击力的数据展示方式

### 学习资源页
- 卡片网格需要重新设计

### 学习路径页
- 时间线展示可以更有动效

## 原则

- 不做花里胡哨，要做有质感的简洁
- 每个页面都要有让人"wow"的细节
- 动效要有目的性（反馈、引导、状态变化），不是为了动而动
- 保持一致性（颜色、字体、间距、圆角）
