## Task 2：课程知识库构建（3天）

### 2.1 课程资料收集
1. 确定目标课程：《机器学习》（推荐）
2. 收集资料来源：
   - 教材PDF（如周志华《机器学习》）
   - 课程PPT（可从学校资源或网络获取）
   - 往年习题集
   - 实验手册与代码示例
   - 学术论文（相关知识点）
3. 资料整理与分类

### 2.2 文档解析
1. 安装解析工具：
   ```bash
   pip install pymupdf pdfplumber python-pptx
   ```
2. PDF解析：
   - 使用pymupdf（即fitz）解析，中文支持更稳定
   - 提取文本内容
   - 保留章节结构
   - 提取图片描述（如有）
3. PPT解析：
   - 提取幻灯片内容
   - 保留层级关系
4. 其他格式解析（Word、Markdown等）

### 2.3 内容分块切割
1. 设计分块策略：
   - 按章节分块（一级分块）
   - 按知识点分块（二级分块）
   - 重叠窗口（保留上下文）
2. 分块参数：
   - 块大小：500-1000 tokens
   - 重叠：50-100 tokens
3. 元数据标注：
   - 章节来源
   - 知识点标签
   - 难度级别

### 2.4 文本向量化
1. 选择Embedding模型：
   - 讯飞星火Embedding API（首选）
   - 或开源模型（如text2vec-base-chinese）
2. 批量向量化处理
3. 向量维度与归一化

### 2.5 向量数据库存储
1. 安装Chroma：`pip install chromadb`
2. 创建知识库Collection
3. 存储向量与元数据
4. 建立索引

### 2.6 RAG检索验证
1. 设计测试查询（20个典型问题）
2. 测试检索效果：
   - 相关性评分
   - 响应时间
   - 召回率
3. 优化分块策略（如效果不佳）

### 2.7 输出物
- [ ] 课程知识库（Chroma数据库）
- [ ] 知识库内容清单（章节、知识点列表）
- [ ] RAG检索测试报告
- [ ] 知识库构建文档（流程、参数、注意事项）

## Task 3：系统架构设计（3天）

### 3.1 多智能体角色定义
1. 画像Agent（Profile Agent）
   - 职责：对话式学习画像构建与更新
   - 输入：学生对话
   - 输出：6维度画像数据

2. 资源Agent群（Resource Agents）
   - 文档Agent：生成课程讲解文档
   - 题库Agent：生成练习题目
   - 思维导图Agent：生成知识点结构图
   - 案例Agent：生成代码实操案例
   - 视频Agent：生成教学视频脚本

3. 路径Agent（Path Agent）
   - 职责：学习路径规划与资源推荐
   - 输入：画像 + 资源元数据
   - 输出：个性化学习路径

4. 评估Agent（Evaluation Agent）
   - 职责：学习效果评估与策略调整
   - 输入：学习行为数据
   - 输出：评估报告 + 调整建议

5. 辅导Agent（Tutor Agent）
   - 职责：即时多模态答疑
   - 输入：学生问题
   - 输出：文字+图解+语音解答

### 3.2 协作流程设计
1. 设计有向状态图（LangGraph）：
   ```
   用户输入 → 画像Agent → 资源Agent群 → 路径Agent → 输出
                ↑                              ↓
                └──────── 评估Agent ←──────────┘
   ```
2. 定义状态流转条件
3. 设计异常处理流程
4. 设计并行执行策略

### 3.3 学习画像6维度模型
1. 知识基础（Knowledge Base）
   - 已掌握知识点
   - 薄弱知识点
   - 知识盲区

2. 认知风格（Cognitive Style）
   - 视觉型/听觉型/动觉型
   - 归纳型/演绎型

3. 学习能力（Learning Ability）
   - 知识吸收速度
   - 理解深度
   - 迁移能力

4. 易错点偏好（Error Patterns）
   - 常见错误类型
   - 错误原因分析

5. 学习目标（Learning Goals）
   - 短期目标
   - 长期目标
   - 职业规划

6. 学习偏好（Learning Preferences）
   - 喜欢的资源类型
   - 学习时间偏好
   - 难度偏好

### 3.4 数据流设计
1. 用户输入数据流
2. Agent间通信数据流
3. 存储数据流
4. 输出数据流

### 3.5 存储方案设计
1. MySQL（关系数据）：
   - 用户表
   - 画像表
   - 资源表
   - 学习记录表
2. Chroma（向量数据）：
   - 知识库
   - 资源向量
3. 阿里云OSS（文件存储）：
   - 生成的文档
   - 视频文件
   - 图像文件
   - 音频文件
   > 注：开发阶段使用本地文件系统，演示/提交时切换为OSS（有学生免费额度）

### 3.6 API接口设计
1. 对话接口（SSE流式）
2. 画像查询/更新接口
3. 资源生成接口
4. 路径查询接口
5. 评估报告接口

### 3.7 输出物
- [x] 架构设计文档（4层架构：前端→Agent编排→模型服务→数据存储）
- [x] 多智能体协作流程图（Orchestrator 意图分类→Agent 调度→SSE 推送）
- [x] ER图（实体关系图）（6张表已实现：users/profiles/resources/paths/records/chat）
- [x] API接口文档（Swagger 自动生成：/docs）
- [x] 6维度画像模型定义文档（student_profiles 表 JSON 字段）
